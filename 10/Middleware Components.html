Middleware Layer

The "Robotics Layer" that abstracts hardware and provides essential services for the flight stack and external communication.

Device Abstraction

Unified driver interface for sensors (IMU, Baro, GPS) and actuators across SPI, I2C, UART, and CAN buses.

System Services

Provides parameter management, persistent storage (dataman), high-rate logging, and the uORB message bus.

External Comms

MAVLink protocol implementation for GCS/Telemetry and APIs for companion computers (ROS/ROS2).

Simulation Bridge

Allows exact same flight code to run on desktop (SITL) controlling a simulated vehicle in Gazebo or JMAVSim.
