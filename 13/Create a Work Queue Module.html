Work Queue Anatomy

Recommended
1

Inheritance Structure

Derive from ModuleBase<T> for lifecycle management and px4::ScheduledWorkItem for scheduling.

2

Queue Selection

Specify queue in constructor (e.g., px4::wq_configurations::lp_default). Must match workload characteristics.

3

Non-Blocking Run()

The Run() method must be fast. No sleep() or blocking I/O allowed. Reschedule at end if periodic.

4

Initialization

Use task_spawn in custom_main to create object, then schedule first run.

Blocking the work queue starves all other tasks on that queue (sensors, estimator, etc.).

Implementation Pattern

C++14
src/modules/my_wq_module/MyWQModule.hpp

// 1. Inherit from ModuleBase and ScheduledWorkItem

class MyWQModule : public ModuleBase<MyWQModule>, public px4::ScheduledWorkItem {

public:

MyWQModule() : ScheduledWorkItem(MODULE_NAME, px4::wq_configurations::lp_default) {}

~MyWQModule() override;

// 2. Implement init() to start scheduling

bool init() {

ScheduleOnInterval(10_ms); // 100Hz

return true;

}

// 3. The work loop (No blocking!)

void Run() override {

if (should_exit()) {

exit_and_cleanup();

return;

}

// Do work here: read sensors, compute, publish

sensor_accel_s data;

if (_sub.update(&data)) {

_pub.publish(data);

}

}

};

For event-driven execution, inherit from px4::ModuleParams and px4::SubscriptionCallbackWorkItem instead.
