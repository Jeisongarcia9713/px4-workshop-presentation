Best Practices & Guidelines

Work Queue vs. Task Selection

Prefer Work Queue for light, periodic, or event-driven tasks to save RAM. Use Tasks only for blocking I/O, long computations, or when strict thread isolation is required.

Performance Optimization

Avoid memory allocations (new/malloc) inside Run() loops. Keep the Run method short and non-blocking to prevent starving other modules on the same queue.

Priority Management

Guard priorities carefully. Don't overuse hp_default work queue unless strictly necessary. Misuse can cause latency spikes in critical flight control loops.

Profiling & Monitoring

Use perf counters to measure execution time. Monitor with uorb top and work_queue status to identify bottlenecks and ensure queue health.
