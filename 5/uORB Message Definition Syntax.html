Message Definition Rules

1

File Structure & Location

Create .msg files in msg/ directory. Use CamelCase for filenames (e.g., VehicleStatus.msg). Must be listed in msg/CMakeLists.txt to be compiled.

2

Mandatory Timestamp

Every message must include uint64 timestamp (microseconds). This is critical for data alignment, logging, and state estimation.

3

Queue & Multi-Topic Config

Optional constants: ORB_QUEUE_LENGTH = N (power of 2) for buffering. # TOPICS a b allows one definition to generate multiple topic IDs.

4

Message Versioning (v1.16+)

For compatibility, use uint32 MESSAGE_VERSION = 1. Place versioned messages in msg/versioned/.
